Function Reference

FileInstall

コンパイル済みスクリプト用にファイルをインクルード、インストールします。

FileInstall ( "source", "dest" [, flag] )

 

パラメータ

source コンパイルするファイルのソースパス。定数文字列でなければいけません。変数、関数の返し値は使用できません。ソースファイル(.au3).に対する相対パス (パス内に .\ や ..\を使用できます)を使用できます。
dest ファイルのディスティネーションパス。ディレクトリのみを指定する場合はバックスラッシュを末尾に付けます。変数を使用できます。 　
flag [オプション] ファイルが既に存在した場合上書きするかどうかのフラグ
  0 = (デフォルト) 上書きしません
  1 = 上書きします

 

返し値

成功 1を返します。
失敗 0を返します。

 

注意

FileInstall関数はコンパイルしたAutoItスクリプトにファイルをインクルードするために設計されました。インクルードされたファイルはコンパイルされたスクリプトの実行時に"取り出"されます。画像などのファイルのインクルードはコンパイルしたスクリプトのサイズを大幅に増加させることに注意してください。
ソースファイル名は変数や計算式、関数呼び出しではなく定数で与えなければなりません ファイルはコンパイル中に特定されなければいけませんが変数や計算式、関数呼び出しではスクリプトがコンパイルされた後、実行されるまで ファイルが特定できないのでソースファイルの定義にには適していません。
またワイルドカードは使用できません。

コンパイルしていないスクリプトでこの関数が使用されると(コンパイル前のテストで使用できるように)コピー操作が代わりにおこなわれます。
インストール時にファイルのもとの作成/更新日時は維持されます。


目的のディレクトリパスはこの関数が呼ばれる時点で存在している必要があります。 さもなければFileInstallは失敗します。その場合、0が返され、ファイルもパスも作成されません。 ディレクトリパスの作成についての情報はDirCreate()を参照してください。

既存ファイルのファイル属性によってはファイルの上書きができないことがあります。 問題なくファイルをインストールできることを保証するためにはFileDelete()またはFileSetAttrib()を使用してください。

 

関連

DirCreate, FileDelete, FileSetAttrib

 

例


; コンパイル時にビットマップ "C:\test.bmp"をインクルードし、実行時に"D:\mydir\test.bmp"にインストールする
$b = True
If $b = True Then FileInstall("C:\test.bmp", "D:\mydir\test.bmp")
